Reto Técnico — IMDUMB
Objetivo: Crear una app iOS llamada IMDUMB que muestre categorías de películas y detalle de cada película. El reto evalúa arquitectura (MVP + Clean), uso de Firebase, consumo de APIs con Alamofire, maquetación con .xib, principios SOLID, uso de Swift Package Manager, y buenas prácticas.
1) Entrega mínima (requerimientos obligatorios)
	•	App iOS en Swift, NO SwiftUI, NO vistas programáticas: toda la UI debe usar *.xib.
	•	MVP + Clean Architecture (separación clara entre layers: View, Presenter, UseCases/Interactors, Repositories, DataStores).
	•	Splash screen que al iniciarse lea datos desde Firebase (elige Realtime Database o Remote Config) y los cargue en memoria (ej: configuración de ambiente, feature toggles, o texto inicial). Mostrar indicador de carga si es necesario.
	•	Pantalla principal: Listado de categorías de películas consumidas desde un API REST usando Alamofire.
	•	Dentro de cada categoría hay películas. El listado de categorías debe usar una UICollectionView. Cada celda de collection view (una categoría) contendrá un UITableView (es decir: TableView dentro de CollectionViewCell) mostrando las películas de esa categoría.
	•	Utilizar Schme o Targets para pequeñas configuraciones
	•	Al seleccionar una película, abrir pantalla de Detalle que incluya al menos:
	•	Carrusel de imágenes (página horizontal con paginación).
	•	Título, calificación, resumen/descripción en HTML.
	•	Lista de actores (nombres). Puede ser un horizontal collection view o lista simple.
	•	Botón fijo inferior "Recomendar".
	•	Al tocar Recomendar: abrir un Modal que:
	•	Muestra el texto del detalle de la película (el modal debe poder crecer en altura según contenido
	•	Incluye una caja de texto para dejar un comentario.
	•	Botón para confirmar; al confirmar, mostrar mensaje de éxito (alert o toast).
	•	Incluir extensiones y protocols para reutilización.
	•	Documentar en el código el uso de al menos 3 principios SOLID en fragmentos concretos (comentarios y pequeñas notas donde se aplican).
	•	Subir proyecto a GitHub/Bitbucket con README (.md) claro de instalación y ejecución.

2) Requerimientos técnicos / sugeridos
	•	Dependencias (SPM): Alamofire (network), opcional: RxSwift (si decide usar programación reactiva). No uses CocoaPods.
	•	Mocking: proveer alternativas de DataStore para Network y Local (p. ej. RemoteDataStore, LocalDataStore, y MockDataStore para tests/manual QA).
	•	Persistencia local opcional: UserDefaults para flags simples; si implementas cache, usa CoreData o Realm (opcional)
3) README.md (qué incluir)
El README del repo debe contener:
	•	Resumen del proyecto.
	•	Tech stack y dependencias (SPM) con versiones.
	•	Cómo correr (Xcode version, pasos: git clone, open IMDUMB.xcodeproj, configurar GoogleService-Info.plist si aplica, compilar).
	•	Endpoint(s) usados o cómo levantar mocks.
	•	Dónde ver la documentación de SOLID
	•	Capturas de pantalla o GIFs (opcionales pero recomendados).
